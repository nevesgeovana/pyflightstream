# TRI-SMI01-CMy: triage of the SMI-01 pitching-moment drift warn (2026-07-21)

## Question

The capstone drift matrix (`DRF-26100-26120_2026-07-21_complete`) flagged
one WARN: the isolated-body pitching moment of SMI-01 (28_B) moved from
CMy -1.7702 on 26.100 (build 5012026) to -1.7843 on 26.120 (build
7012026), about 0.8 percent. Solver change, case-setup sensitivity, or
noise? (PLN-013; first real finding of the SMI drift class, HND-015.)

## Method

1. Repeatability: one fresh SMI-01 run per version, same explicit
   executables as the capstone, scratch workroot, reports committed as
   `PHY-26100_2026-07-21_smi01-triage-rerun` and
   `PHY-26120_2026-07-21_smi01-triage-rerun`.
2. Input identity: sha256 of the opened .fsm compared across both runs.
3. Boundary localization: per-boundary loads of both runs compared from
   the exported spreadsheets (local only; this note carries aggregated
   values, CLAUDE.md invariant 5).

## Evidence

* Both reruns are bit-identical to the first measurements: 26.100
  reproduces CL -0.0181, CDi 0.0030, CDo 0.1311, CMy -1.7702; 26.120
  reproduces CL -0.0174, CDi 0.0030, CDo 0.1305, CMy -1.7843. Noise is
  excluded on both sides.
* Both runs opened the same file: sha256
  `43b2373b316a552abc665eae052f7f174e36d5da7455e267efbe3a152f49d401`.
  Identical script grammar for both versions (the 26.100 backfill found
  the grammars identical, HND-014). Case-setup sensitivity is excluded.
* The 28_B model exposes a single boundary, so the movement localizes
  trivially to the isolated body. CL and CDo move together with CMy
  (all deterministically), while the full configuration (SMI-02) stayed
  inside 0.35 percent in the capstone matrix.

## Conclusion

A deterministic solver-side change between builds 5012026 and 7012026
shifts the isolated-body solution slightly (CMy -0.8 percent, with
small consistent movements in CL and CDo). This is exactly the
sensitivity the SMI drift class exists to surface (SAD Section 11).

## Decision

The SMI-01 reference stays untouched: it was seeded on 26.120 and
encodes current-version behavior. The cross-version WARN stands by
design, documenting the real movement between builds; no band
recalibration. PLN-013 closed.
